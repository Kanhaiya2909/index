Git Command

Command Description
git init The git init command creates a new Git repository.
git status The git status command displays the state of the working directory and the staging area.
git add {fileName} Staged particular file. eg: {fileName}
git add -A to the staging area (All Files & Folder)
git commit -m captures a snapshot of the project's currently staged changes.
git checkout {particularFile} to copy the file.
git checkout -f to copy all the file.
git log displays all of the commits in a repository's history.
git log -p -1 displays last commits in a repository's history in full details
git commit -a -m skipping the stage area.
git rm --cached {fileName} To remove from staging area.
git rm -f {fileName} To delete that particular file.
git branch display all available branches
git branch {fileName} create new branch 'fileName'
git checkout {branchName} To update branch/ Jump to another branch
git merge {branchName1} branchName1 will merge with current branch.
git checkout -b {branchName} Automatically creat and update the branch
git stash Save changes in local memory
git stash list To display all changes by stash command
git stash apply {stash@{0}} To update file
git stash drop {stash{0}} to delete files...
git stash clean to delete all stash files.
git log --pretty=oneline to display id in one line
git revert {id} to remove wrong commit
git reset --soft {id} --> soft/hard/mixed/merge/keep undo
